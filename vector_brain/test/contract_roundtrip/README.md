# Cross-language contract round-trip

The seed of the P1 golden-vector parity harness. Proves `schema/engram.fbs` compiles and
round-trips **identically across Python and C++** — the load-bearing claim behind choosing
FlatBuffers for the EGRV cert (see `curation/vector_brain_stack.md` §3).

**Flow:** `flatc` generates C++ + Python from the schema → `write_cert.py` writes an EGRV
buffer (`egram.bin`) → `read_cert.cpp` reads it back and asserts every field.

## Run
```bash
apt-get install -y flatbuffers-compiler libflatbuffers-dev   # flatc + C++ headers
pip install flatbuffers
./run.sh
```
Expected tail: `PASS: cross-language contract round-trips`

Generated code (`VectorBrain/`, `engram_generated.h`), `egram.bin`, and the `read_cert`
binary are build artifacts — git-ignored, regenerated by `run.sh`.

> Verified 2026 with flatc 2.0.8 / g++ 13.3: Python wrote `person=dexter sense=Meaning
> repr=rfft.embed.v1 dim=4 data=1..8`; C++ read all fields back identically.
